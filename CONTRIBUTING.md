# Contributing

## Workflow

All contributions must come through a fork and pull request — direct pushes to `main` are not allowed.

1. Fork this repository
2. Create a branch off your fork (`git checkout -b fix/your-topic`)
3. Make your changes
4. Open a pull request against `main` here
5. Wait for review — PRs require at least one approval before merge

## Guidelines

- Keep changes focused and small — one concern per PR
- Don't add dependencies unless necessary; this extension has none by design
- Test against a live pi install before submitting
- Commit messages should be plain and descriptive

## Reporting issues

Open a GitHub issue. Include pi version, OS, and the exact error or behavior you saw.
